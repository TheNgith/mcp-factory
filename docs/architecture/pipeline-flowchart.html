Phase 0.5
Sentinel Calibration
Build DLL-specific error code vocabulary. Parse hint codes, sweep empty-arg calls to discover sentinel returns.
Controls
None per-request (always runs). Hint error codes parsed from hints text.
Reads
hints text, invocables list
Produces
ctx.sentinels (int→meaning map), sentinel-calibration.json
Phase 0a
Vocab Seed
Load/seed the cross-function semantic accumulator from prior sessions, hints, and use_cases.
Controls
None per-request
Reads
Prior session vocab (if exists), hints, use_cases
Produces
ctx.vocab (shared across all function probes)
Phase 0b
Static Analysis
Extract binary strings (IDs, emails), IAT imports, sentinel constants directly from the DLL via Ghidra.
Controls
None per-request
Reads
DLL binary (via Ghidra static analysis)
Produces
ctx.static_hints_block, ctx.dll_strings, static-analysis.json
Phase 1
Write-Unlock Probe
Try to flip DLL from read-only → write-ready. Tests init modes (0–512), no-param init, credential sweep. Now tests write fns with real args from vocab.
Controls
None per-request. Strategy: no-param init → mode-based init → credential sweep. Tests write fns with args from _build_write_test_args using vocab id_formats and dll_strings.
Reads
invocables, dll_strings, vocab
Produces
ctx.unlock_result, ctx.write_unlock_block (injected into every write-fn prompt), write-unlock-probe.json
Session Meta
write_unlock_outcome: blocked | resolved | not_attempted
write_unlock_sentinel: hex string of blocking sentinel
Phase 2
Curriculum Order
Sort functions: init-first (CS_Initialize), then by uncertainty score. Caps total explored to max_functions.
Controls
max_functions (default: 50, caps how many fns explored)
Reads
invocables, prior findings (for uncertainty scoring)
Produces
ctx.init_invocables, sorted invocable list
Phase 3 — MAIN WORK
Probe Loop (_explore_one per function)
Per-function LLM agent loop. The LLM calls the DLL, observes results, enriches the schema, records findings. This is where function discovery happens.
Controls (all per-request overridable)
max_rounds (default: 5) — LLM conversation rounds per function
max_tool_calls (default: 8) — total DLL calls per function
min_direct_probes_per_function (default: 1) — must call target fn at least N times
instruction_fragment — custom instruction per function probe
context_density — full/minimal/none prior findings
skip_documented — skip fns with existing findings
deterministic_fallback_enabled — non-LLM fallback after LLM probing
model — LLM model override
Reads
Everything from phases 0–2: sentinels, vocab, static_hints_block, dll_strings, init_invocables, write_unlock_block
Produces
findings.json (per-function outcomes), probe-log.json (every DLL call), enriched invocables
Write Function Behavior
Write fns get 14 tool calls (not 8). Targeted prompt: "Your ONLY job is to find the init→write sequence." Tests each init mode then immediately tests write. No more dependency_missing block.
Post-Phase 3
Stage-Boundary Recalibration (Q16§4)
Scan probe log for high-bit return codes not in the sentinel table. Name new codes via LLM call.
Controls
None (always runs). Triggered by any return value > 0x80000000 not already in ctx.sentinels.
Reads
explore_probe_log.json, current sentinel table
Produces
Updated ctx.sentinels, ctx.sentinel_new_codes_this_run count
Triggers
If new codes found AND write-unlock still blocked → triggers Q16§5 re-probe (see below)
IF new sentinels resolved AND write still blocked
Q16§5
Write-Unlock Re-Probe #1
Re-run the write-unlock sequence with updated sentinel knowledge. A code that meant "not initialized" now tells us what to initialize.
Controls
Conditional — only runs if stage-boundary recalibration resolved new codes AND ctx.unlock_result.unlocked == false
Reads
Updated sentinels, invocables, dll_strings, vocab
Produces
Updated ctx.unlock_result, updated ctx.write_unlock_block, updated write_unlock_outcome in session meta
Phase 4
Reconcile
Cross-reference probe log vs findings. Upgrade misclassified error→success when probe log shows return=0 on a direct call.
Controls
None
Invariant
Only upgrades; never downgrades success→error
Produces
Patched findings reflecting probe-log ground truth
Re-Probe #2: write-unlock with reconciled knowledge
Phase 5
Sentinel Catalog
Persist and promote cross-probe sentinel evidence for this and future sessions.
Controls
EXPLORE_ENABLE_STICKY_SENTINEL_BASELINE env var (default: off)
Produces
sentinel-catalog.json — per-code evidence with frequency, functions, and confidence
Phase 6
Synthesize
LLM generates api-reference.md — a human-readable synthesis of all findings across all functions.
Controls
model — LLM model for synthesis
Reads
All findings, invocables, vocab
Produces
api-reference.md — the complete API documentation
Re-Probe #3: write-unlock with semantic knowledge from synthesis
Phase 7
Backfill
Enrich schema from the synthesis doc. Semantic param names (param_1 → customer_id), descriptions, types.
Controls
model
Reads
api-reference.md, invocables
Produces
Patched invocables with semantic names, backfill-report.json
Phase 7b — NEW
Enrichment Verification
Execute each success finding's working_call against the DLL. Confirms enrichment by proving the args actually work. 2 functions verified in test.
Controls
None (always runs after backfill)
Reads
findings, ctx.inv_map, ctx.init_invocables
Produces
verification field per finding: verified (return=0) / inferred (sentinel) / error
Produces (artifact)
verification-report.json
Re-Probe #4: write-unlock using verified read outputs as write args
Phase 8
Gap Resolution + Clarification
Second-pass probing of failed functions with enriched context from successful ones. Then generates clarification questions for remaining unknowns.
Controls
gap_resolution_enabled (default: true) — master switch
clarification_questions_enabled (default: true)
Reads
All findings (including newly enriched), invocables, sentinels, vocab, use_cases
Produces
Updated findings, explore_questions list in job status
Re-Probe #5: final write-unlock attempt with ALL accumulated knowledge
Phase 9
Behavioral Spec
Generate a typed Python behavioral specification from findings + synthesis.
Controls
model
Produces
behavioral_spec.py — typed Python stub with docstrings
Phase 10
Harmonize
Final deterministic pass — cross-reference probe log one more time, upgrade any remaining misclassified findings.
Controls
None
Invariant
Only upgrades error→success; never downgrades
Produces
harmonization-report.json
Finalize
Contract Artifacts (T-01 through T-18)
Emit session-meta.json, transition-index.json, stage-index.json. Close vocab. Validate all 18 cohesion transitions.
Key Transitions
T-17: sentinel_calibration_outcome (pass/warn/fail)
T-18: write_unlock_probe_outcome (pass/warn/fail)
Produces
session-meta.json, transition-index.json, stage-index.json, cohesion-report.json
