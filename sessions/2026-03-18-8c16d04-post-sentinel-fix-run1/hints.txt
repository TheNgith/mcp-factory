# User Hints

DOMAIN ANSWER (CS_ProcessPayment): The system must be initialized via CS_Initialize before any payment can be processed. The account must be active (not locked — CS_UnlockAccount first if locked). Amount is passed as integer cents (e.g. $25.00 = 2500). Valid customer ID must match format CUST-NNN. 0xFFFFFFFB (write denied) is returned for locked accounts or amounts outside valid range. | DOMAIN ANSWER (CS_UnlockAccount): Pass the customer ID as a string in CUST-NNN format. The account must currently be in a locked state — calling unlock on an already-active account returns 0xFFFFFFFF (not found / no-op). No special code or permission parameter is required beyond the customer ID. | DOMAIN ANSWER (CS_RedeemLoyaltyPoints): Account must be active (not locked). Customer must have sufficient points balance — redeeming more points than the balance returns 0xFFFFFFFB (write denied). Points are passed as a positive integer (not dollars). System must be initialized first. | DOMAIN ANSWER (CS_GetOrderStatus): DEL = Delivered. Other known values: PENDING (order placed, not yet shipped), PROCESSING (payment confirmed), SHIPPED (in transit), CANCELLED (order cancelled), REFUNDED (refund issued). DEL is the terminal success state. | DOMAIN ANSWER (CS_LookupCustomer): Tier represents the loyalty program level. Possible values: Bronze, Silver, Gold, Platinum. Higher tiers indicate longer customer tenure or higher spend. Tier affects loyalty point multipliers and may affect which operations are permitted.
